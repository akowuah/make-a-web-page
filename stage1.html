LESSON 1 NOTES

WHAT TO KEEP (from lectures)

1 - What a Web Page is

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.


2 - How Coding Works

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote.


3 - Computers are Stupid

Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax"). For example, if you forget to close a < b > tag, the computer won't be able to figure out what you had intended to make bold. This "stupidity" can be very frustrating, but it also gives programmers incredible power: if you know how to talk to a computer than you can tell it to do anything you want.


4 - Programmers Can't Remember Everything

There are too many details to keep everything in your head. And that's okay. If you forget how to make text italic in HTML, you can always just look it up


5 - Basic HTML Vocabulary

You will be using HTML in the next few lessons, so it will be helpful if you're comfortable with the jargon.
  • Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag < p > for example and a closing tag, < /p >. Some tags (called "void" tags) do not require a closing tag (like the < br > tag).
  • Element: An HTML element refers to everything within a set of opening and closing tags.
  • Attribute: This is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">

1. The Web:

A collection of HTML docs. Glued together by links. Major players are: You, Internet, Web Servers. Traffic is transferred using a protocol called HTTP. You make request through browser to retrieve files from servers.


2. HTML:

The heart of the web; Made up of:
  - txt content
  - markup (plain text is plain text in html. To let plain text look different, you need to use markup)
  - references to other documents
  - links to other pages


    2a. Markup

Consists of elements. Elements are made up of opening and closing tags and the substance in between them.

  Tags:

  • < b > tag: it bolds
  • < strong > tag: similar to bold
  • < em > tag: it italicizes
  • < form > tag: creates form for user input

    2b. HTML Attributes < tag attr = “value”> contents < /tag >
  • Anchor tag: Used for making links <a href=www.reddit.com> derp </a>
  • Image tag: Used for referencing images <img src= “url” alt = “text”>
    NOTE: Image tags have no closing tags. It’s called a void tag (i.e. no closing tag).

Another tag without a closing tag is <br>. The break tag is used to move content to the next line. The <p> </p> tag also moves content to another line. It defines paragraphs (a block, not just an inline). The p tag is NOT a void tag.


    2c. Container tags
  • Span tag: <span> </span>; span is a block tag (creates a block or space around the element.
  • Div tag: <div> </div>; div is an inline tag (does not create a block around the element)

LESSON 2 NOTES

HTML is like a house with STRUCTURES/rooms; CSS is the STYLING that is added to the house. There are RULES to follow when using HTML and TOOLS to help use HTML. One good way to review HTML is using chrome developer tools. Developer tools is accessible by pressing f12 in chrome. Websites have a tree-like structure. By clicking on elements of the tree, e.g. h1, img etc in google chrome developer mode, you can view more details about the elements such as pix size and border size of images.

The important things to take away are:

  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks)
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page


How the browser knows:

HTML: HTML is the language of the internet. The HTML language has specific rules that need to be followed just like typical languages have grammatical rules. HTML describes the structure and content of the webpage. The building blocks of HTML is a “tag”. The browser turns tags into elements. An element begins with a starting tag and end with an ending tag. Everything between the tags is called content. The elements form a tree.

Tags Each tag has a:
  1. name e.g. <h1>. The name tells us what kind of element will be created (e.g. a paragraph, block, image etc.)
  2. some have attributes with values e.g. <li class= “article”>


CSS: CSS is another language. It defines the style of a webpage e.g. size of font color, background, borders, positions etc.


To tie structure (i.e. HTML) and style (i.e. CSS), HTML must tell the browser what style to use and where to get it.
Once you understand the rules that govern how these languages work, you can look at an HTML code and get a pretty good understanding of how a webpage is laid out.

Note: Everything on webpages are actually boxes square spaces. We create boxes using the div tab.
